Secure Baselines are standard security configurations applied to systems, networks, or applications to guarantee minimum security.

üîë Establish:
Create a baseline by identifying required security controls and configurations.
  • Examples: Defining password policies, disabling unnecessary services, enforcing encryption, setting logging requirements.

    • üîë Deploy:
    Apply secure baseline to systems, applications, or devices.
    - Examples: Firewalls, user permissions, encryption protocols, up-to-date antivirus.
    - Automation Tools: Group Policy (Windows), Ansible, SCCM, imaging.
    - Cloud Example: AWS Config for defining and deploying secure configurations across cloud resources.

    üîë Maintain:
    Continuously monitor and update baselines to address new vulnerabilities, patches, and organizational changes.
  • Examples: Updating baseline after critical OS patches or new compliance requirements.


    • üß© Hardening Targets:
    • Mobile devices: MDM, encryption, screen locks, remote wipe.

    • Workstations: Disable unused ports/services, apply patches, enforce least privilege.

    • Switches: Enable port security, disable unused ports, configure VLANs.

    • Routers: Disable default accounts, secure management interfaces, ACLs.

    • Cloud Infrastructure: Enforce IAM policies, restrict storage bucket access, logging.

    • Servers: Patch OS, enforce access control, enable host-based firewalls.

    • ICS/SCADA: Segregate networks, restrict external access, patch cautiously.

    • Embedded Systems / RTOS: Lock firmware, restrict external ports, minimal code, isolate tasks.

    • IoT Devices: Change default credentials, update firmware, segment networks.


üéØ Goal:
Ensure systems are configured securely, reduce vulnerabilities, and maintain consistent protection across all computing resources.
