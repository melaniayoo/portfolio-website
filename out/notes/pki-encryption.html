üîê Cryptography & PKI


    • Cryptography: Study of codes to protect information.

    • Public Key Infrastructure (PKI): System managing asymmetric key pairs (public/private) for secure communication, authentication, and encrypted data transfer.

    • Public Key Cryptography: Asymmetric encryption for encrypting and decrypting messages.

    • Symmetric Encryption: Uses a single key (AES, DES, 3DES, Twofish, Blowfish, RC4/5/6).

    • Hybrid Encryption: Asymmetric encryption used to transfer symmetric keys securely.

    • Stream & Block Ciphers: Encrypt bit-by-bit or in fixed-size blocks.

    • Key Management Tools: TPM, HSM, secure enclaves.


    üõ°Ô∏è Data Protection Techniques


    • Tokenization: Substitute sensitive data with non-sensitive tokens.

    • Data Masking: Disguise sensitive data while keeping it usable.

    • Hashing: One-way cryptographic function producing a fixed-length digest (MD5, SHA-1/2/3, RIPEMD, HMAC).

    • Salting & Key Stretching: Strengthen passwords against cracking.

    • Pass the Hash & Birthday Attacks: Common hash-based attacks.

    • Hash digest encrypted with sender's private key.

    • Ensures integrity, authenticity, and non-repudiation.

    • Used in code signing and Digital Security Standard (DSS).

    üìú Certificates


    • Digital Certificates: Bind public key to identity.

    • Certificate Authorities (CAs): Issue and validate certificates.

    • Registration Authority (RA): Collects info for certificates.

    • Types: Wildcard, SAN, single-sided, dual-sided, self-signed, third-party.

    • Trust & Revocation: Root of trust, CRLs, OCSP, OCSP Stapling.

    • Key Escrow & Recovery Agents: Manage lost or secure keys.
