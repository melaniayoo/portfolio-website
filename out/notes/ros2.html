    üîß Core ROS2 Concepts


    Nodes


    • Independent subprograms performing specific tasks

    • Communicate through topics, services, and parameters

    • Can be written in Python or C++


    Python Template:

    Topics


    • Asynchronous message-passing system

    • Publishers send messages

    • Subscribers receive them

    • Nodes do not need direct knowledge of one another


    Services


    • Synchronous client‚Äìserver communication

    • Clients send requests; servers send back responses

    • Ideal when a response is required


    Custom Interfaces


    • Creating custom .msg and .srv types

    • Defining dedicated interface packages

    • Configuring CMakeLists.txt and package.xml for builds


    Parameters


    • Node configuration at runtime

    • Must be declared before use

    • Can be set via command line or launch files


    Launch Files


    • Start multiple nodes together

    • Support XML and Python formats

    • Essential for complex systems and simulations


    üõ†Ô∏è Key ROS2 Tools & Commands


    • `ros2 run` ‚Äì Run executables

    • `ros2 node list/info` ‚Äì Inspect nodes

    • `ros2 topic list/echo/info` ‚Äì Debug topics

    • `ros2 service list/call` ‚Äì Service tools

    • `ros2 interface show` ‚Äì Inspect message/service types

    • `ros2 param list/get/set` ‚Äì Parameter tools

    • `ros2 bag record/play` ‚Äì Record/playback data

    • `rqt_graph` ‚Äì Visualize node communication

    • `rqt` ‚Äì GUI tool suite


    üß™ Development Workflow


    • Create workspace: `mkdir -p ros2_ws/src`

    • Create packages: `ros2 pkg create`

    • Write nodes using Python/C++ templates

    • Add dependencies to package.xml and CMakeLists.txt

    • Build: `colcon build`

    • Source environment: `source install/setup.bash`

    • Run and debug your nodes
